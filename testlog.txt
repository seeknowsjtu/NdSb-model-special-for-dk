[Interpretation note]

The latest run improves the global cost numerically, but the fitted pulse_width (~1.17 ps) is clearly inconsistent with the experimental time resolution (~150 fs). Therefore this improvement is not fully trustworthy physically.

Current interpretation:
1. raw_m_chi2q remains the best large-scale readout model.
2. The optimizer is using an unphysically broad pulse to smooth over the early-time mismatch.
3. The next step is not to add more free parameters immediately, but to constrain the pulse width using experimental knowledge.

Next planned run:
- keep observable_mode = raw_m_chi2q
- fix pulse_width = 1.5e-13 s
- fix B1_obs = 0
- fit only [S_scale, A_obs, B0_obs, G_es0]
- then re-evaluate whether the early-time mismatch persists

Decision rule:
- If the early-time mismatch remains large even with a physically fixed pulse width, then the model likely needs an additional prompt electronic readout term rather than another slow dynamical parameter.
